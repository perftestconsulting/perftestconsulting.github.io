Introduction to the Training Course

This course is designed for quality assurance professionals who want to gain knowledge and hands-on experience in software performance testing. Throughout this course, you will learn the fundamental principles and advanced techniques of software performance testing. Starting from the basics, I'll guide you step-by-step through the process of setting up a performance testing process from scratch for a real web application.

The training course is suitable for engineers across all levels, from junior to senior QA.

Module 1: Performance Testing Basic Concepts.

In this module, we delve into the fundamental aspects of performance testing. This module introduces essential terminologies, principles, metrics and characteristics for performance assessments.

- Why does performance matter? Real case studies of huge impact of poor performance on software companies.

- Definition of performance testing.

- Performance testing types with real examples.

- Performance and resources utilization metrics and characteristics.

Module 2: Project(release) assessment Phase.

In this module, we explore project assessment phase that occurs before setting up performance testing. This phase involves a thorough evaluation of the project to understand its non functional requirements, objectives, and potential challenges, ensuring that performance testing is effectively aligned with the project goals.

- The goals of project release assessment phase.

- Performance assessment checklist.

- Non functional requirements(NFR) related to performance: definition, best practices and anti patterns.

- NFR templates and definition guide.

Module 3: Performance test planning.

This module covers a creation of performance test plan that outlines the approach, test types, resources, and schedule for the performance testing activities of a project. It serves as a roadmap for the testing team and provides a clear framework for executing performance tests.

- Goals of planning phase.

- Creation of a performance test plan. Content of performance test plan.

- Performance test plan best practices and recommendations.

- Template of performance test plan.

Module 4. Scripting.

In this module, we focus on the creation and customization of performance test scripts, which are essential for simulating user behavior and interactions with the application. The Scripting module provides practical insights into writing, optimizing, and managing test scripts to ensure they accurately represent real-world scenarios and generate reliable performance data.

- Type of workloads and workload modeling.

- Performance test cases as basis for scripts.

- Popular performance testing tools: JMeter, LoadRunner, Gatling, etc. Performance test tool selection principles.

- Scripting:

-- What is a performance test script?.

-- Script structure: initialization section, main body, cleanup

-- Recording vs. hand-coding scripts.

-- Best practices: parameterization, correlation, think times and pacing, assertions, error handling, no resource demanding e.t.c

-- Data driven concepts.

-- Debugging.

-- Scripting checklist.

-- Common pitfalls and anti patterns in test scripts development.

Module 5: Apache Jmeter Hands-On Workshop.

This hands-on workshop ensures that participants acquire practical, real-world skills in using Apache JMeter for scripting and effective performance testing.

- Overview of Apache JMeter.

- Overview of sample web application we will create performance test for.

- Definition of non functional requirements for the sample web app.

- Sample web application workload modeling.

- Creation of performance test cases.

- Development of Apache Jmeter test script for the sample web app.

- Applying scripting best practices.

- Performance test execution.

- Monitoring.

- Basic results analysis.

Module 6: Test execution.

In this module, we focus on the execution phase of performance testing, where planned tests are run to gather critical performance data. The Test Execution module covers the procedures, tools, and best practices required to execute performance tests effectively, ensuring that the results are accurate and meaningful.

- Preparing for test execution.

- Running a test. We will use the performance test created during Apache Jmeter workshop.

- Baseline and benchmarking concepts.

- Troubleshooting. Handling errors and failures.

- Test execution best practices.

Module 7: Results analysis and tuning.

Topics in this module include data interpretation, trend recognition, performance metrics, and iterative optimization techniques.

- Collecting Results.

- Key principles to results analysis.

- Key metrics to analyze.

- Results analysis: patterns, bottlenecks.

- Tuning Strategies.

Module 8: Reporting.

You will learn how to structure information effectively, use visual aids for clarity, and communicate findings clearly and persuasively. Emphasis is placed on organizing data, highlighting key findings, and crafting actionable recommendations.

- What stakeholders want.

- What stakeholders (usually) get.

- Best Practices for reporting.

- Valuable charts and their uses.

- Common mistakes in preparing charts.

- Key elements of a performance test report.

- Key elements of performance bug report.

Conclusion

In the conclusion, we will review what you have learned throughout the course and discuss the next steps for applying your new knowledge.
